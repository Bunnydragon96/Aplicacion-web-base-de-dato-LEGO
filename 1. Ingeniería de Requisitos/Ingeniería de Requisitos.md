# Estudio de factibilidad 
El proyecto consiste en desarrollar una aplicación web para gestionar e interactuar con una base de datos de conjuntos de LEGO, utilizando un conjunto de datos públicamente disponible en Kaggle. Desde el punto de vista **técnico**, el proyecto es viable dado que se dispone de los datos en formato estructurado (CSV) y las herramientas necesarias para su transformación, almacenamiento y presentación son ampliamente documentadas, de uso libre y compatibles con el entorno académico; el equipo cuenta además con los conocimientos adquiridos durante el curso para llevar a cabo las tareas de modelado, normalización e implementación requeridas. En cuanto a la **factibilidad económica**, el costo es nulo o mínimo, ya que todos los recursos necesarios son de distribución libre y gratuita, sin dependencia de licencias comerciales. **Operativamente**, el proyecto es manejable dentro del tiempo disponible para el semestre: las etapas están claramente delimitadas y el alcance es acotado, con entregables concretos como los diagramas, el esquema relacional, las consultas y la interfaz de visualización. Por último, desde la perspectiva **legal**, el conjunto de datos proviene de una fuente pública con acceso abierto, por lo que no existen restricciones de uso que impidan su utilización con fines educativos. En conclusión, se recomienda proceder con el desarrollo del proyecto, ya que cumple satisfactoriamente con los criterios de factibilidad técnica, económica, operativa y legal.

# Captura de requisitos

La captura de requisitos es basada en los requisitos proveídos por el profesor de la clase.


# Definición de requisitos

## Requisitos funcionales


| RF # | Definición                                                                                   |
| ---- | -------------------------------------------------------------------------------------------- |
| 1    | La aplicación web permitirá al usuario crear una cuenta nueva.                               |
| 2    | La aplicación web autenticará al usuario antes de entrar a la página.<br>                    |
| 3    | la aplicación web visualizará una subconsulta con o son WITH                                 |
| 4    | La aplicación web visualizará una consulta que muestra todas las tablas en la base de datos. |
| 5    | La aplicación web visualizará una consulta que permita escoger un atributo de una relación.  |
| 6    | La aplicación web visualizará un JOIN de 3 tablas.                                           |
| 7    | La aplicación web visualizará una consulta con agregación.                                   |
| 8    | La aplicación web permitirá al usuario insertar datos.                                       |
| 9    | La aplicación web permitirá al usuario modificar datos.                                      |

## Requisitos no-funcionales

| RNF # | Definición                                                                                                     |
| ----- | -------------------------------------------------------------------------------------------------------------- |
| 1     | La aplicación web debe ser alojado en un servidor dedicado que permita su acceso remoto a través del internet. |
| 2     | La aplicación web debe contar con un dominio propio que permita ser accedida a traves del internet.            |
| 3     | La aplicación web debe ser accesible desde los principales navegadores web.                                    |
| 4     | La aplicación web debe ser intuitiva, si no, que provea instrucciones.                                         |
